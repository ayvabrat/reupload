Какие данные собрать, чтобы сгенерировать промпт для Lovable + VPS + эта админка
================================================================================

Пришлите (можно одним сообщением) — тогда можно собрать готовый промпт с подставленными значениями:

1) Сайт Lovable
   • URL опубликованного фронта (например https://xxxx.lovable.app или свой домен).
   • Стек, если знаете (React/Vite и т.д.) — опционально.

2) VPS / бэкенд
   • Публичный IP или домен, куда будет смотреть фронт (например api.mysite.ru).
   • ОС (Ubuntu 22.04 и т.д.).
   • Будет ли HTTPS только на nginx/Caddy или уже есть.

3) База данных на VPS
   Вариант A — оставить SQLite у ReUpload Detector (проще):
   • Абсолютный путь к файлу на сервере, например: /var/lib/reupload/database.db
   • Пользователь ОС, от которого крутится uvicorn (имя пользователя).

   Вариант B — PostgreSQL/MySQL отдельно:
   • Тип СУБД, хост, порт, имя БД, пользователь, пароль.
   • (Важно: текущий проект из коробки использует SQLite; перенос на Postgres потребует доработки кода и миграций — в промпте это нужно явно указать.)

4) Доступ к админке ReUpload Detector после переноса
   • Уже задано в .env: WEB_AUTH_USER=admin, WEB_AUTH_PASSWORD=123456 (смените на проде).
   • Порт приложения, например 8765, и будет ли он за reverse proxy (nginx → 127.0.0.1:8765).

5) CORS / домены
   • Точный origin фронта Lovable (например https://app.lovableproject.com), если фронт и API на разных доменах — понадобится настроить CORS на FastAPI (сейчас allow_origins=["*"] — часто хватает; для cookie/credentials может понадобиться точный origin).

6) Что именно «перенести»
   • Только доступ к данным с Lovable через API к этому серверу, или
   • Встроить iframe со старой панелью, или
   • Полностью повторить UI в Lovable и дергать REST этого проекта — уточните желаемое поведение.

---

Готовый шаблон промпта (подставьте свои значения вместо <...> и отдайте нейросети / разработчику)
--------------------------------------------------------------------------------------------------

Скопируйте блок ниже, замените плейсхолдеры и при необходимости дополните.

"""
Контекст: есть бэкенд ReUpload Detector (Python FastAPI, uvicorn), раздаёт API /api/* и собранную админку из web/dist. Авторизация веб-панели — HTTP Basic Auth, логин <WEB_AUTH_USER>, пароль <WEB_AUTH_PASSWORD>. БД SQLite по пути <DB_PATH_ON_VPS> на VPS <VPS_IP_OR_DOMAIN>.

Задача:
1) Развернуть на VPS этот бэкенд (uvicorn, systemd, при необходимости nginx с SSL), чтобы API было доступно по <PUBLIC_API_URL> (например https://api.example.com).
2) Фронтенд сделан в Lovable, публичный URL: <LOVABLE_FRONT_URL>.
3) Настроить так, чтобы <описание: например «страница админки в Lovable открывала ту же панель по ссылке» или «Lovable вызывал API по базовому URL ... с Basic Auth в заголовках»>.
4) Убедиться, что данные (SQLite файл или указанная БД) лежат на постоянном томе и переживают перезапуск.
5) Документировать переменные окружения: WEB_AUTH_USER, WEB_AUTH_PASSWORD, DB_PATH, WEB_API_HOST=0.0.0.0, WEB_API_PORT, и как проксировать порт через nginx.

Ограничения: текущий проект использует SQLite через SQLAlchemy; без отдельной миграции кода Postgres не подключать.
"""

---

Примечание: пароль 123456 в открытом виде в репозитории и в чатах не публикуйте; на продакшене замените на сложный и храните только на сервере.
