Развёртывание на сервере (сайт + логин + сохранение данных)
============================================================

Это не статическая HTML-страница, а серверное приложение (Python + FastAPI).
Его запускают на VPS/хостинге с Python или в Docker; «сайт» — это та же панель,
отдаваемая сервером.

Вход по логину
--------------
В .env на сервере:

  WEB_AUTH_USER=admin
  WEB_AUTH_PASSWORD=123456

Пока WEB_AUTH_USER непустой, браузер запросит логин/пароль (HTTP Basic Auth).
Смените пароль после первого входа. Для HTTPS используйте nginx/Caddy с
сертификатом — иначе пароль передаётся по сети в открытом виде.

Отключить пароль (только локально): закомментируйте или очистите WEB_AUTH_USER.

Путь /api/health без пароля, если WEB_AUTH_ALLOW_HEALTH=true (по умолчанию).

Где хранятся данные
-------------------
• SQLite: переменная DB_PATH в .env.
  На сервере задайте абсолютный путь на диск с постоянным томом, например:

  DB_PATH=/var/lib/reupload/database.db

• Экспорты: EXPORT_DIR (например /var/lib/reupload/exports).
• Настройки Telegram/мониторинга: data/app_settings.json рядом с рабочим
  каталогом (или задайте каталог через расположение проекта).

Создайте каталоги и права на запись пользователю, от которого крутится uvicorn.

Слушать интерфейс сервера
-------------------------
В .env:

  WEB_API_HOST=0.0.0.0
  WEB_API_PORT=8765

Перед сайтом обычно ставят nginx как reverse proxy на 127.0.0.1:8765 и SSL.

Пример systemd (упрощённо)
--------------------------
WorkingDirectory=/opt/reupload-detector
EnvironmentFile=/opt/reupload-detector/.env
ExecStart=/opt/reupload-detector/.venv/bin/uvicorn web_dashboard:app --host 0.0.0.0 --port 8765

После правок: npm run build в web/, затем перезапуск сервиса.
